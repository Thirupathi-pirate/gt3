The actual exploit procedure

Prerequisites

  1. A Windows PC (or Linux with udev rules)
  2. USB data cable (original preferred)
  3. Phone battery >60% charged
  4. Confirmed storage type (eMMC or UFS) from Step 1

Unlock Steps

  1. Download and extract CVE-2022-38694_Unlock_v1.72.zip
  2. Navigate to the folder matching your storage type:
    • eMMC: ums9230_universal_emmc/
    • UFS: ums9230_universal_ufs/
  3. Power OFF the phone completely
  4. Run the unlock batch script as Administrator
  5. Hold Vol Up + Vol Down on the phone and connect the USB cable
  6. The tool will detect the device in EDL mode and execute the exploit
  7. Wait for the "Unlock successful" message
  8. Disconnect and reboot the phone
⚠️

If the script says "ERROR: FDL mismatch" or similar, DO NOT continue. You're using the wrong files. Disconnect immediately and re-check your storage type.

💡

Refer to the README in the tool for OS-specific instructions. Linux users need to set up udev rules for USB access.

🤖 Flashing a GSI

Realme C71 has no device-specific custom ROMs. Your only option is GSI (Generic System Image) — a system image that works across Treble-compatible devices.

Recommended: LineageOS 22.2 GSI

Android 15 QPR2, most mature GSI with best Unisoc support.

Download
github.com/MisterZtr/LineageOS_gsi
→ LineageOS-22.2--GAPPS-EXT4.7z
Least buggy choice

Latest: LineageOS 23.2 GSI

Android 16 QPR2, newest but fewer Unisoc-specific patches.

Download
github.com/MisterZtr/LineageOS_gsi
→ LineageOS-23.2--VANILLA-EXT4.7z
Newest, more bugs likely

Fastboot Flash Commands

Fastboot GSI Flash
# Boot to fastboot
adb reboot bootloader

# Wipe system
fastboot erase system

# Flash GSI (adjust filename)
fastboot flash system lineage-22.2-*-EXT4.img

# Wipe data (factory reset)
fastboot -w

# Reboot
fastboot reboot
💡

Important: Always use the EXT4 variant first — it's fully read-write and most compatible. Only try EROFS if you're sure your kernel supports it (kernel 5.4+).

Known Bugs by Category

🔴 High Impact

  • RIL (no mobile network) — Try: settings put global restricted_networking_mode 0 or flash stock vendor
  • Camera not connecting — Set persist.camera.HAL3.enabled=1 in build.prop + permissive SELinux
  • VoLTE / ViWiFi — Almost never works on GSIs. Requires OEM IMS blobs.

🟡 Medium Impact

  • Brightness slider broken — Fixed by mikasagsifixed Magisk module
  • WiFi not working — Try phh-treble settings to set WiFi interface manually
  • 120Hz refresh rate loss — May default to 60Hz; check phh-treble settings
  • Fingerprint sensor — Unisoc FP HAL is vendor-specific; usually doesn't work

🟢 Low Impact

  • DT2W (double tap to wake) — Rarely works; vendor-specific driver
  • BT audio codecs — Usually works but may have codec issues with LDAC/aptX
  • SEAndroid — Must be permissive for many features (breaks SafetyNet)

🔵 Fixable

  • mikasagsifixed module fixes brightness slider, device spoofing, SELinux rules
  • phh-treble settings app (included in many GSIs) has toggles for WiFi, Bluetooth, display
  • GApps vs Vanilla — VANILLA variants tend to have fewer compatibility issues

🛡️ mikasagsifixed — GSI Bug Fix Module

A Magisk module designed for Unisoc T616/T7250 devices to fix common GSI issues. Install after booting into a GSI for the first time.

✅ What It Fixes

  • Brightness slider — Sets max backlight value properly
  • Device spoofing — Sets correct system properties for GSI compatibility
  • SELinux rules — Permissive policies for system access
  • Build fingerprint — Updated Android 13 fingerprint for better app compatibility

❌ What It Does NOT Fix

  • RIL (mobile network / data)
  • WiFi / Bluetooth hardware issues
  • Camera HAL problems
  • Fingerprint sensor
  • DT2W / VoLTE / ViWiFi
  • 120Hz refresh rate

Installation

  1. Download the latest mikasagsifixed.zip from GitHub
  2. Boot into your GSI and complete initial setup
  3. Transfer the ZIP to your phone
  4. Open Magisk app → Modules → Install from storage
  5. Select the ZIP → Reboot

🆘 Hard Brick Recovery

If your C71 is bricked (no boot, no recovery):

  1. Install SPD UpgradeDownload Tool v1.2012+ on Windows
  2. Find the EDL test points on the motherboard (search "Realme C71 EDL test point" on GSM Helpful)
  3. Short the test points and connect USB — device enters EDL mode
  4. Flash the stock .pac firmware via SPD tool
  5. If EDL mode isn't detected, you may need authorized Realme service center
⚠️

Brick recovery requires disassembling the phone to access EDL test points. If you're not comfortable with hardware, seek professional help.

🔧 Troubleshooting

Exploit script shows "FDL mismatch" or "ERROR"

Wrong storage type selected (eMMC vs UFS mismatch)
  • ✅ Disconnect immediately — do NOT continue
  • ✅ Re-run lsblk on your phone to confirm storage type
  • ✅ Download and use the correct folder (ums9230_universal_emmc or ums9230_universal_ufs)
  • ✅ If unsure, check the scatter file from your stock ROM

Phone not detected in EDL mode

Missing USB drivers / wrong button combo / bad cable
  • ✅ Install Unisoc USB drivers on Windows (search "Unisoc SPRD driver")
  • ✅ Ensure phone is fully powered OFF (hold Power 10s)
  • ✅ Connect USB while holding Vol Up + Vol Down
  • ✅ Try a different USB cable and USB 2.0 port

GSI boots but no mobile network (RIL dead)

Vendor RIL blobs incompatible with GSI
  • ✅ Try: adb shell settings put global restricted_networking_mode 0
  • ✅ In phh-treble settings: set "RIL implementation" to "AOSP"
  • ✅ Flash stock vendor partition back: fastboot flash vendor vendor.img (extract from stock ROM)
  • ✅ If nothing works, RIL is likely broken on this GSI — try another

Brightness slider doesn't work

GSI doesn't have proper backlight driver integration
  • ✅ Install mikasagsifixed Magisk module (most reliable fix)
  • ✅ In phh-treble settings → "Force brightness" → set max value
  • ✅ Or add persist.sys.phh.max_brightness=255 to build.prop

Camera shows black screen or won't open

HAL3 not enabled / SELinux blocking camera HAL
  • ✅ Add to build.prop: persist.camera.HAL3.enabled=1
  • ✅ Set SELinux to permissive: setenforce 0 (or via Magisk module)
  • ✅ Try OpenCamera app instead of stock camera
  • ✅ If still broken, camera may not work on any GSI for this device

WiFi won't turn on / can't find networks

WiFi HAL interface mismatch
  • ✅ In phh-treble settings: set "WiFi interface" to wlan0
  • ✅ Try flashing stock vendor partition
  • ✅ Check if MAC address shows as 00:00:00:00:00:00 — if so, WiFi HAL is broken

120Hz display stuck at 60Hz

GSI defaults to 60Hz refresh rate
  • ✅ In phh-treble settings → "Force FPS" → set 120 (or "Highest")
  • ✅ Or use ADB: settings put global peak_refresh_rate 120
  • ✅ May cause instability — test before relying on it

Bootloop after flashing GSI

Wrong GSI variant / dirty flash / incompatible partition
  • ✅ Boot to recovery → Format data → Reboot
  • ✅ If still bootlooping: fastboot erase system → reflash with EXT4 variant
  • ✅ Make sure you selected A-only or A/B variant correctly (C71 uses A/B slots)
  • ✅ Last resort: restore stock firmware via SPD tool

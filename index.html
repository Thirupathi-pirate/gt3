1
Disable Windows Defender
Required — unsigned drivers won't install otherwise
🔴

This is mandatory. If you skip this, you'll get "Waiting for device" errors and the unlock will fail.

  1. Open Windows Security → Virus & threat protection
  2. Click Manage settings under "Virus & threat protection settings"
  3. Toggle Real-time protection → OFF
  4. Search for Core Isolation → Memory Integrity → Toggle OFF
  5. Restart your PC — important, drivers won't load without reboot
✅

Expected: After reboot, Windows will show a shield with a red X in the system tray — this means Defender is disabled. You can re-enable it after the unlock is complete.

3
Install USB Drivers (Penumbra)
Enables communication between PC and phone in preloader mode
  1. Navigate to C:\gtneo3-unlock
  2. Right-click PenumbraDrivers.exe → Run as Administrator
  3. In the driver installer window, select libusb as the driver type
  4. Click Install and wait for completion
  5. If Windows shows a security warning, click "Install anyway"
  6. Restart your PC — the drivers will not work without a reboot
💡

If the installer says "no devices found" — that's normal. The driver gets installed regardless. The phone will be detected when you connect it in Step 5.

5
Flash ENG Preloader
The critical step — bypasses bootloader locks
  1. Power OFF your phone completely
  2. Wait 10 seconds after the screen goes black
  3. On your PC, run unlockbt1.bat
  4. The script will display: Waiting for device...
  5. Now connect the phone to the PC via USB cable
  6. Do NOT press any buttons — the phone should be off
  7. The script will detect the phone and flash the ENG preloader automatically
  8. When complete, the phone will boot into Recovery Mode

⚠️

Script stuck on "Waiting for device"? Try these fixes in order:

  1. Hold Vol Up + Vol Down while connecting the USB cable
  2. Use a USB 2.0 port (black color, not blue)
  3. Try a different USB cable (must be data sync, not charge-only)
  4. Reinstall PenumbraDrivers (Step 3) and reboot
  5. Double-check Windows Defender is OFF (Step 1)
💡

Touch not working in recovery? This happens if you have a replaced/non-original screen. Connect a USB OTG mouse via a USB-C adapter to navigate.

6
Format Data in Recovery
Required after flashing the ENG preloader
  1. Your phone should now be in Realme Recovery Mode (blue/orange background)
  2. Use Volume buttons to navigate, Power button to select
  3. Select Format data from the menu
  4. Type yes when prompted (use the on-screen keyboard)
  5. Wait for the format to complete (takes 10–30 seconds)
  6. Select Reboot
  7. The phone will restart and boot into Android Setup
💡

Pro tip: You don't need to sign in to Google or connect to WiFi yet. Skip through the setup wizard quickly — you'll set it up properly after the unlock is complete.

7
Enable OEM Unlocking
Must be ON for the unlock command to work
  1. Phone will boot to the Android setup screen
  2. Skip through the setup — tap "Skip" or "Not now" for everything
  3. Go to Settings → About Phone
  4. Tap Build Number 7 times quickly (you'll see a toast: "You are now a developer!")
  5. Go back → Settings → System → Developer Options
  6. Find OEM Unlocking and toggle it ON
  7. Also toggle USB Debugging ON
🔴

If OEM Unlocking is greyed out or missing, your slot might still be on B, or the ENG preloader didn't flash correctly. Do NOT proceed — go back to Step 5.

Rooting with Magisk
Based on XDA Root Guide

Step 1: Get boot.img

You need the stock boot image from your current firmware to patch with Magisk. Choose either option:

🅰️ Extract from OTA

Install payload_dumper and extract from your firmware package:

Terminal
pip install payload_dumper
payload_dumper.py ota.zip
🅱️ Pull from Phone

Extract boot.img directly from your device via ADB:

ADB
adb shell
dd if=/dev/block/bootdevice/by-name/boot of=/sdcard/boot.img
adb pull /sdcard/boot.img

📌 Your firmware OTA URL: https://gauss-opexcostmanual-eu.allawnofs.com/remove-08a77f8628e6aa653cb262be7656d279/component-ota/25/03/25/d0d6022e9eeb44c7b5b91bc343f93fa0.zip

Step 2: Patch with Magisk App

Magisk modifies the boot image to grant root access. Follow these steps on your phone:

  1. Transfer boot.img to your phone (USB or cloud)
  2. Download the latest Magisk APK
  3. Install the APK on your phone
  4. Open Magisk app → tap Install next to "Magisk"
  5. Tap "Select and Patch a File"
  6. Navigate to and select your boot.img
  7. Wait for patching to finish (5–10 seconds)
  8. Transfer magisk_patched-*.img back to your PC
🛡️

The patched file is usually named
magisk_patched-xxxxxxxx.img

Step 3: Flash Patched Boot & Verify

Boot your phone into fastboot mode (Power + Vol Down) and connect to PC:

Fastboot
fastboot flash boot magisk_patched-*.img
fastboot reboot

After reboot, open the Magisk app. You should see:

✅

Installed: Yes (with version number)

If it doesn't show "Installed: Yes", re-check the filename and try again.

🔄 How to Apply OTA Updates After Unlock (Stock + Rooted)

You can still receive OTA updates with an unlocked bootloader. To keep root after updating:

  1. When OTA is downloaded, do NOT press "Install & restart" yet
  2. Open Magisk app → tap Install → Install to Inactive Slot
  3. Magisk will root the other slot before the OTA applies
  4. After Magisk finishes, go back and tap "Install & restart" in the OTA dialog
  5. Phone reboots, applies OTA, and you keep root automatically
💡

After major Android upgrades (e.g. Android 14 → 15), you may need to re-patch boot.img manually. Always check XDA for compatibility.

🔧 Troubleshooting

"Waiting for device" — script can't detect phone

Windows Defender blocking drivers / wrong USB port / bad cable / wrong phone mode
  • ✅ Disable Windows Defender (Real-time + Core Isolation), then reboot
  • ✅ Use a USB 2.0 port (black, not blue)
  • ✅ Try a different data cable (not charge-only)
  • ✅ Hold Vol Up + Vol Down while connecting
  • ✅ Reinstall PenumbraDrivers → reboot PC
  • ✅ Check Device Manager for "libusb" or "MediaTek" devices

Recovery boots but touch doesn't work

This happens if you have a replaced or non-original screen
  • ✅ Connect a USB OTG mouse via USB-C adapter
  • ✅ Navigate to "Format data" using the mouse, then reboot
  • ✅ If no OTG adapter, try pressing Power 10s to force reboot

"flashing unlock is not allowed"

OEM Unlocking is disabled in Developer Options
  • ✅ Boot to system normally (not bootloader)
  • ✅ Settings → Developer Options → Enable OEM Unlocking
  • ✅ adb reboot bootloader then fastboot flashing unlock
  • ✅ If OEM Unlocking is greyed out, the ENG preloader didn't flash fully — retry Step 5

Phone stuck at logo after unlock (won't boot)

Normal after unlock — data needs to be formatted
  • ✅ Boot to Recovery: Power off → hold Power + Volume Down
  • ✅ Select Format data → type yes → reboot
  • ✅ If recovery doesn't load, try Power + Volume Up instead

"No valid slot to boot"

Boot partition pointer is corrupted or on wrong slot
  • ✅ fastboot --set-active=a then fastboot reboot
  • ✅ If that fails: fastboot --set-active=b then try again
  • ✅ Boot to recovery → Format data

OTA download gives 403 error

Realme's Gauss servers block direct browser downloads
  • ✅ Install 1DM (Download Manager) from Play Store
  • ✅ Copy the OTA link into 1DM — it handles the required headers
  • ✅ Alternative: download via romdevelopers.com mirror

Hard brick — phone completely dead, no response

Flashed preloader_raw.img, cross-flashed wrong firmware, or relocked bootloader on non-stock
  • 1️⃣ Hold Vol Up + Vol Down → connect USB → check Device Manager for "MediaTek USB Port"
  • 2️⃣ If detected → use SP Flash Tool or MTKClient to restore stock firmware
  • 3️⃣ If not detected → needs Oplus Auth Tool (service center level)
  • 4️⃣ Get help: Telegram Support Group
  • 5️⃣ Last resort: authorized Realme service center (may charge)

Lost fastboot mode after successful unlock

Rare — fastboot driver issue or boot partition issue
  • ✅ Re-run the unlock procedure from Step 5 (flash ENG preloader again)
  • ✅ Or restore backed-up preloader via MTKClient
  • ✅ On PC: reinstall fastboot drivers (part of platform-tools)

Phone boots but no network / no IMEI

NV data partition was wiped during flashing
  • ✅ Restore from the backup you made in Step 4 (see NV Restore section above)
  • ✅ If no backup → permanent loss, send to service center

Fastboot commands not recognized (PC side)

platform-tools not in PATH or wrong directory
  • ✅ Ensure you're in C:\gtneo3-unlock\platform-tools or have added it to PATH
  • ✅ Run: cd C:\gtneo3-unlock\platform-tools first
  • ✅ Or use full path: C:\gtneo3-unlock\platform-tools\fastboot.exe

Phone won't enter preloader mode

Phone not fully powered off / wrong button combination
  • ✅ Hold Power 10+ seconds to force shutdown
  • ✅ Wait 15 seconds after screen goes black
  • ✅ Try connecting with no buttons pressed first
  • ✅ Then try Vol Up + Vol Down held while connecting
